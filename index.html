cookieless_sessions by tatyree

Rails sessions without cookies.

We had finally learned to stop worrying and love cookie-based sessions on Rails. Then we developed our first cross-platform site for the UK mobile market. Suddenly, we were scrabbling around in a dark world of closed, proprietary, cookie-swallowing gateways, phone browsers developed by standards fundamentalists, and “smart phones” made from 10-year-old innards stuffed in new cases with colour screens and extra memory soldered on. Hair and sleep were lost trying to make phones on one very large network log in. On another very large network, we could get them to log in, but only long enough to make two, maybe three requests before the session was lost.

The problem was the cookie: Lots of phones don’t like them (and don’t let anyone tell you different). Very large network number one doesn’t like them at all. Very large network number two has some gateways that are OK with cookies and others that strip them. The only sensible thing to do was to implement cookieless sessions. Since developing the first site, we have been working on several others, hence the existence of this gem: no more copy-and-pasting monkey patches between projects.

If you use it and like it then, please, go out and pet a kitten (apologies to Hampton Catlin). If you don’t, then complaints to the usual place.
